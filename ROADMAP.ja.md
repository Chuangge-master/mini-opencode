# ロードマップ

このドキュメントは、mini-OpenCode の開発計画と今後の機能について概説しています。コミュニティからの貢献を歓迎します！

## 🚀 計画中の機能

### コア機能
- [ ] **エージェントランタイムサンドボックス**: エージェントが生成したコードを安全かつ隔離された環境で実行するためのサンドボックス環境を実装します。
- [ ] **確認メカニズム**: ユーザーの指示が曖昧または不完全な場合に、エージェントが自発的に確認の質問を行えるようにします。
- [ ] **長期記憶**: エージェントがセッションをまたいでプロジェクトのコンテキスト、ユーザー設定、履歴を記憶できるようにする永続的な記憶機能を実装します。

### UI/UX の改善
- [ ] **サブエージェントの可視化**: 複雑なタスク委任時のサブエージェントの階層と活動を可視化する UI コンポーネントを追加します。
- [ ] **推論プロセスの表示**: 透明性を向上させるために、モデルの内部的な「思考」または「推論」プロセス (Chain-of-Thought) の表示をサポートします。
- [ ] **TUI パフォーマンスの最適化**: ターミナルユーザーインターフェース (TUI) のレンダリングエンジンを最適化し、遅延を減らして応答性を向上させます。

---

これらの機能の実装にご協力いただける場合は、ガイドラインについて [CONTRIBUTING.md](./CONTRIBUTING.md) をご覧ください。
